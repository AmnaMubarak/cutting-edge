Kottler Metals Vendor Statement Automation

Comprehensive documentation for automating vendor statement processing with Microsoft Business Central integration

Overview

This documentation outlines the automated process for handling vendor statements at Kottler Metals, featuring direct integration with Microsoft Business Central.

Key differentiator: Unlike previous implementations, this system integrates directly with Microsoft Business Central's PO module instead of JobBoss2.

Technical Architecture

The technical implementation follows a modular architecture with distinct components handling specific responsibilities.

The system is built with error handling and retry mechanisms at each step to ensure robust processing.

System Components

Email Receiver

  • Monitors KottlerAP@kottlermetals.com
  • Filters relevant vendor invoices
  • Forwards to BlueOceanAP@BlueOceansp.ai
  • Maintains email threading

Email Parser

  • Extracts attachments
  • Validates file formats
  • Identifies invoice types
  • Prepares for OCR processing

OCR Service

  • Performs text extraction
  • Identifies key fields
  • Maps data to schema
  • Validates extraction quality

Process Steps

1. Email Reception and Validation

  • System monitors dedicated email inbox
  • Validates sender and email format
  • Checks for required attachments
  • Initiates processing workflow

2. Document Processing

  • Extracts attachments from email
  • Validates file formats (PDF, Excel, etc.)
  • Prepares documents for OCR
  • Maintains audit trail

3. OCR and Data Extraction

  • Performs OCR on documents
  • Extracts key fields:
    • Invoice Number
    • Date
    • Amount
    • PO Reference
    • Vendor Details
  • Validates data quality
  • Flags issues for review

4. MBC Integration

  • Queries MBC API for PO validation
  • Matches invoice to PO data
  • Validates amounts and details
  • Posts matched invoices
  • Handles exceptions and errors

5. Document Management

  • Stores processed documents in SharePoint
  • Follows naming convention: Customer\AP invoices\Year\Vendor\Month\invoice
  • Maintains version control
  • Ensures audit compliance

6. Error Handling and Notifications

  • Implements retry mechanisms
  • Sends notifications for:
    • Processing errors
    • Validation failures
    • Manual review requirements
    • Successful processing
  • Maintains error logs
  • Provides error resolution paths
