HighGate Invoice Processing System

An automated solution for processing Oracle PPR (Payment Process Request) batches through web automation

Overview

The HighGate Invoice Processing System automates the end-to-end process of submitting, managing, and tracking Payment Process Request (PPR) through Oracle ERP. The system eliminates manual data entry, reduces processing time, and provides comprehensive tracking and reporting capabilities.

Key Benefits

  • Automated batch processing of invoice data
  • Integration with SharePoint for file storage and retrieval
  • Comprehensive status tracking and reporting
  • Robust error handling and recovery mechanisms
  • Email notifications for process status updates

Step 1: Payment File Upload

The system detects new payment files in SharePoint and validates their format for processing.

  • SharePoint file detection
  • File format validation
  • Initial data loading

Step 2: Batch Data Extraction

Excel files are parsed to extract batch details and normalized for Oracle submission.

  • Excel parsing & data normalization
  • Batch preparation for Oracle
  • Data validation

Step 3a: Oracle Login & PPR Submit

The system logs into Oracle and submits PPR details through the web interface.

  • SSO Authentication
  • Submit PPR page navigation
  • PPR detail entry & submission

Step 3b: Manage & Resume PPR

The system navigates to Manage PPR module to review installments and initialize approval.

  • Manage PPR page navigation
  • Installment validation & review
  • Resume PPR & approval initiation

Step 4: Status Reporting & Monitor

The system tracks batch status and generates reports on approval progress.

  • Batch status tracking
  • Approval verification
  • Report generation and notification

Error Handling and Resilience

Error Type Handling Strategy
Navigation Failures Retry with exponential backoff
Element Not Found Fallback to alternative XPaths/IDs
Batch/Invoice Mismatches Report discrepancy and continue with next batch
Network Timeouts Retry with increased timeouts
Authentication Failures Relogin and retry

Enhanced Navigation Resilience

The system implements a robust retry mechanism for navigation operations:

  • Multiple retry levels (outer loop for entire process, inner loop for individual links)
  • Exponential backoff (5s, 10s, 20s) between retry attempts
  • Alternative clicking methods (JavaScript fallback)
  • Page refreshes between retry attempts
  • Detailed logging of retry operations
